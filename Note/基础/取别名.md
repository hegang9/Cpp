# 取别名

C++ 中有三种主要方式为已有类型取别名。

## 方式一：宏定义

### 特点

- 只在预编译阶段进行文本替换
- 无类型检查
- 在连续声明时可能发生错误

### 语法

```
#define (你想要的别名) (原名字)
```

### 用法 1：对象式

```cpp
#define PI 3.14159
```

### 用法 2：模拟函数行为（警惕使用）

```cpp
#define MAX(a, b) ((a) > (b) ? (a) : (b))
```

## 方式二：typedef

### 特点

- 在编译阶段为已有类型别名
- 支持类型检查
- 受类/命名空间作用域限制

### 语法

```
typedef (原名字) (你想要的别名)
```

### 示例

```cpp
typedef int integer;
typedef struct { int x; int y; } point;
```

## 方式三：using（C++11 开始使用）

### 特点

- 模板友好
- 在编译阶段为已有类型别名
- 支持类型检查
- 受类/命名空间作用域限制

### 语法

```
using (你想要的别名) = (原名字)
```

### 示例

```cpp
using IntPtr = int*;
```

## 对比总结

| 特性 | 宏定义 | typedef | using |
|-----|-------|---------|-------|
| 类型检查 | ✗ | ✓ | ✓ |
| 作用域限制 | ✗ | ✓ | ✓ |
| 模板友好 | ✗ | ✗ | ✓ |
| 预处理时机 | 编译前 | 编译时 | 编译时 |
