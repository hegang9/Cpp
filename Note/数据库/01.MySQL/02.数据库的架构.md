# 02.数据库的架构

数据库架构通常包括三层：

1. 外层模式（外部架构）：也称为用户视图，是用户对数据库的逻辑观察方式，定义了用户能够看到和使用的数据。
2. 概念层（概念架构）：也称为全局视图或逻辑架构，描述整个数据库系统中数据的总体结构和关系。它表示对应用程序员或系统分析师来说最重要的内容。
3. 内层模式（内部架构）：也称为存储视图或物理架构，它描述了数据在计算机上实际存储的方式。涉及到磁盘存储、文件组织、索引方法等底层操作。

这种三层结构的设计方式被称为 ANSI/SPARC 三级体系结构模型。在实际应用中，可以根据具体需求增加或减少这些层次。

除此之外，还有一些其他类型的数据库架构：

1. 主从复制架构：将主数据库中的所有更改同步到一个或多个备份数据库中。
2. 分布式数据库架构：将数据分散到多个不同地理位置或处理器之间以提高性能和可靠性。
3. 数据库集群技术：通过共享硬件资源和负载均衡技术将多个独立的计算机组成一个集群，提高数据库系统的可扩展性和容错能力。
4. 数据库镜像架构：将主数据库的所有更新复制到一个或多个备份数据库上，以提高数据的可靠性和安全性。
